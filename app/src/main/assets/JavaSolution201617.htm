5. Why Java is a Strongly Typed Language?

Answer: First, every variable has a type, every expression has a type, and every type is strictly defined. Second, all assignments, whether explicit or via parameter passing in method calls, are checked for type compatibility. There are no automatic coercions or conversions of conflicting types as in some languages. The Java compiler checks all expressions and parameters to ensure that the types are compatible. Any type mismatches are errors that must be corrected before the compiler will finish compiling the class.

[A]Write a multithreaded program to display numbers from 1 to 100. Odd numbers displayed by Main Thread and Even numbers are displayed by Child thread. Numbers should be displayed in sequence.

Answer:

class PrintNumber

{

boolean mainThread=true; synchronized void printOdd(int i)

{

if(mainThread==false) try{wait();}catch(Exception e){}

System.out.println(i);

mainThread=false;

notify();

}

synchronized void printEven(int i)

{

if(mainThread==true) try{wait();}catch(Exception e){}

System.out.println(i);

mainThread=true;

notify();

}

}

class T extends Thread

{

PrintNumber number;

T(PrintNumber number)

{

this.number=number;

}

public void run()

{

for(int i=2;i<=100;i=i+2) number.printEven(i);

}

}

class M

{

public static void main(String s[])

{

PrintNumber number=new PrintNumber(); T childThread=new T(number); childThread.start();

for(int i=1;i<100;i=i+2) number.printOdd(i);

}

}
